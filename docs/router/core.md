# Core Router

The old core router page has been split into smaller pages:

- [Route model](/router/routes) describes route state, params, `beforeOpen`,
  parent routes, virtual routes, and chained routes.
- [Router and history](/router/router) describes route registration, history
  adapters, scopes, activation flow, and nested routers.
- [Navigation](/router/navigation) describes `route.open`, `router.navigate`,
  links, redirects, and boundary usage.
- [Query tracking](/router/query-tracking) describes query-driven dialogs,
  filters, exits, and manual checks.

This split mirrors Virentia core docs: one page per mental pattern, with each
page explaining the problem, API shape, common cases, and where the boundary
belongs.
