# Storage

`@virentia/storage-core` сохраняет сторы Virentia в подключаемые бэкенды — `localStorage`, `sessionStorage`, query-строку URL, память или ваш собственный — и держит их синхронизированными **в обе стороны**.

Состояние остаётся моделью: стор помнит значение, а привязка `persist` отражает это значение в **бокс** для одного scope. Модель по-прежнему не знает, где она рендерится и где хранится.

```sh
pnpm add @virentia/storage-core @virentia/core
```

## Боксы

**Бокс** — небольшой бэкенд, из которого `persist` читает и в который пишет. Пять встроенных:

| Бокс | Бэкенд | Переживает | Внешняя синхронизация |
|------|--------|-----------|------------------------|
| `local()` | `localStorage` | перезагрузки, перезапуски | другие вкладки (`storage`) |
| `session()` | `sessionStorage` | перезагрузки (на вкладку) | — |
| `query()` | URL `?key=value` | в URL / истории | назад / вперёд (`popstate`) |
| `memory()` | `Map` в процессе | сессию | тот же процесс |
| `custom(impl)` | всё, что вы дадите | зависит от вас | зависит от вас |

DOM-боксы падают в `memory()`, когда окружение недоступно (SSR, воркеры) или заблокировано (приватный режим), так что один код модели работает на сервере без изменений. См. [Боксы](/ru/storage/boxes) про каждый, сериализацию и поведение при SSR.

## Сохранение стора

`persist` держит один writable-стор и один бокс синхронизированными для одного scope:

```ts
import { scope, scoped, store } from "@virentia/core";
import { local, persist } from "@virentia/storage-core";

const theme = store<"light" | "dark">("light");
const app = scope();

scoped(app, () => {
  persist({ source: theme, key: "theme", storage: local() });
});
```

При первом запуске стор заполняется из бокса, если ключ есть; дальше каждое изменение записывается наружу, а внешнее изменение (другая вкладка, назад/вперёд) подтягивается обратно. Защита от повторного входа рвёт петлю write↔watch, так что значение из бокса не пишется в него сразу же обратно.

Про scope, владельцев и свою сериализацию — см. [Сохранение](/ru/storage/persist).
